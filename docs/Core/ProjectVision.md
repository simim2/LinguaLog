# Project Vision

## LinguaLog

**Version:** 1.0

---

# 1. Vision

LinguaLog는 단순한 AI 영어 일기 웹사이트가 아닙니다.

LinguaLog의 목표는 **사용자의 영어 성장 과정을 기록하고 분석하는 플랫폼**을 만드는 것입니다.

영어를 얼마나 잘하는지를 평가하는 것이 아니라,

시간이 지나면서 사용자의 영어가 어떻게 변화하고 성장하는지를 기록하고 시각화하는 것이 핵심 목표입니다.

---

# 2. Mission

영어를 꾸준히 공부하는 것은 어렵습니다.

많은 학습 서비스는 문제를 풀거나 점수를 제공하지만,

실제 사용자가 어떤 영어를 사용했고 어떻게 성장했는지는 보여주지 못합니다.

LinguaLog는

* 영어를 기록하고
* AI가 분석하고
* 성장 데이터를 축적하고
* 변화 과정을 시각화

하는 것을 목표로 합니다.

---

# 3. Core Values

## 3.1 Record First

가장 중요한 것은 꾸준히 기록하는 것입니다.

AI 분석보다 먼저

사용자가 부담 없이 영어를 작성할 수 있어야 합니다.

모든 기능은

"더 많이 기록하도록 돕는가?"

를 기준으로 설계합니다.

---

## 3.2 Growth Over Perfection

완벽한 영어보다

성장을 중요하게 생각합니다.

하나의 문장이 틀렸는지보다

3개월 전보다 얼마나 좋아졌는지가 더 중요합니다.

---

## 3.3 Data-Driven Learning

모든 AI 분석 결과는

단순히 화면에 보여주기 위한 정보가 아닙니다.

향후

* Dashboard
* Growth Report
* Statistics
* Mobile App

에서 재사용 가능한 데이터로 저장합니다.

---

## 3.4 Long-term Consistency

새로운 기능을 추가할 때마다

기존 구조를 유지하면서 확장 가능하도록 개발합니다.

빠른 개발보다

유지보수성을 우선합니다.

---

# 4. Product Goal

LinguaLog는 다음 질문에 답할 수 있어야 합니다.

* 나는 영어를 얼마나 자주 쓰는가?
* 어떤 주제로 영어를 많이 사용하는가?
* 내 문법은 얼마나 발전했는가?
* 내 어휘 수준은 어떻게 변화했는가?
* 어떤 상황에서 영어를 가장 많이 사용하는가?
* 지난달보다 영어가 얼마나 성장했는가?

---

# 5. Target Users

주요 대상은

* 영어 일기를 쓰는 학습자
* 영어 회화를 연습하는 학습자
* 꾸준히 영어를 기록하고 싶은 사용자

입니다.

시험 점수보다

실제 영어 사용을 중요하게 생각하는 사람을 대상으로 합니다.

---

# 6. Product Philosophy

LinguaLog는 AI가 주인공인 서비스가 아닙니다.

주인공은 사용자입니다.

AI는

사용자의 영어를 평가하는 존재가 아니라,

성장을 도와주는 조력자입니다.

따라서 AI 분석은

간결하고 이해하기 쉬워야 하며,

사용자를 위축시키지 않아야 합니다.

---

# 7. Long-term Roadmap

## Phase 1

기본 영어 일기 작성

## Phase 2

AI 분석

* CEFR
* Topic
* Feedback

## Phase 3

분석 데이터 확장

* Conversation Type
* Keywords
* Grammar
* Vocabulary

## Phase 4

Dashboard AI Insights

* Conversation Type Distribution
* Keyword Ranking
* Grammar Trend
* Vocabulary Trend

## Phase 5

Search & Filter

* 제목 검색
* 키워드 검색
* Conversation Type 검색
* CEFR 검색

## Phase 6

Growth Report

* 월별 성장 분석
* 영어 사용 패턴
* AI 리포트

## Phase 7

Cloud Sync

* 사용자 계정
* 데이터 동기화

## Phase 8

Flutter Mobile App

웹과 동일한 데이터 구조를 사용하는 모바일 애플리케이션 개발

---

# 8. Technical Philosophy

프로젝트는 다음 원칙을 따릅니다.

* 기존 구조를 최대한 유지한다.
* 최소한의 수정으로 기능을 확장한다.
* AI Provider를 교체할 수 있도록 추상화한다.
* UI와 AI 로직을 분리한다.
* Dashboard는 저장된 데이터를 활용한다.
* 모든 데이터는 모바일에서도 재사용 가능하도록 설계한다.

---

# 9. Success Criteria

LinguaLog가 성공했다고 판단하는 기준은

* 사용자가 매일 영어를 기록한다.
* 영어 성장 데이터를 확인할 수 있다.
* Dashboard가 의미 있는 통계를 제공한다.
* 사용자가 자신의 영어 변화를 직접 느낄 수 있다.

---

# 10. Future Vision

궁극적으로 LinguaLog는

"AI 영어 일기"

가 아니라

**"나의 영어 성장 기록"**

이 되어야 합니다.

사용자는 언제든 자신의 과거 영어를 돌아보고,

현재와 비교하며,

앞으로의 목표를 세울 수 있어야 합니다.

모든 기능은 이 비전을 중심으로 설계합니다.
